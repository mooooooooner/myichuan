﻿# Magai 靶场研究交接文档

## 1. 研究目标与结论概览

本次针对 `https://beta.magai.co` 的研究，已确认两条核心链路：

1. **会话续期链路（Supabase）**：`refresh_token -> access_token` 可行。
2. **聊天调用链路（Magai 业务层）**：`next-action` 先签发短 JWT，再调用 `/api/chat`。

关键结论：

1. `Supabase access_token` **不能直接**调用 `https://beta.magai.co/api/chat`。
2. `/api/chat` 需要 **Magai 自签短 JWT**（约 5 分钟有效）。
3. 这个短 JWT 由 `next-action` 请求返回，已定位到 action id：
   - `40cd8b2ec4704e0f3c267bd98f93b0f9806e121b77`

## 3. Chat 主链路（已打通）

### 3.1 前置：创建与占位

前端先在 Supabase 写入 `spark` 占位记录（`chat_generated_text: <TYPING>`），再关联到 `chat_context`。

典型请求：

1. `POST /rest/v1/spark`（创建占位）
2. `POST /rest/v1/chat_context_context_messages_rel`（关联关系）

### 3.2 换取 Magai 短 JWT（核心）

请求：

1. `POST https://beta.magai.co/`
2. Header 包含：
   - `next-action: 40cd8b2ec4704e0f3c267bd98f93b0f9806e121b77`
   - `accept: text/x-component`
   - `content-type: text/plain;charset=UTF-8`
   - 需要登录态 `cookie`

响应（RSC 格式）第二行返回短 JWT，例如：

```text
1:"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

该 JWT 特征：

1. payload 含 `random / iat / exp`
2. 有效期约 `300s`

### 3.3 调用 `/api/chat`

请求：

1. `POST https://beta.magai.co/api/chat`
2. `Authorization: Bearer <短JWT>`
3. Body 关键字段：
   - `model`
   - `modelId`
   - `chatId`
   - `sparkId`
   - `messages`
   - `skipTitleGeneration`
   - `enabledTools`
   - `enabledToolkits`
   - `userTimezone`

实测返回：

1. `200 OK`
2. `Content-Type: application/x-ndjson; charset=utf-8`
3. 流式事件：`start -> text-delta -> response-metadata -> finish`

示例片段：

```json
{"type":"start"}
{"type":"text-delta","text":"..."}
{"type":"response-metadata","modelId":"anthropic/claude-4.6-sonnet-20260217"}
{"type":"finish","finishReason":"stop"}
```

### 3.4 回填消息与 usage

生成完成后，前端会 `PATCH /rest/v1/spark?id=eq.<sparkId>`，回填：

1. `chat_generated_text`
2. `chat_json`
3. `word_usage`
4. `chat_reasoning`

样本中：

1. `word_usage = 41`
2. 与回复词数一致（按词统计，不是 token 数）

## 5. Refresh Token 续期链路

### 5.1 已确认接口

```http
POST https://bkatrpghmzbpjhegvkev.supabase.co/auth/v1/token?grant_type=refresh_token
Content-Type: application/json
apikey: sb_publishable_xxx
authorization: Bearer sb_publishable_xxx

{"refresh_token":"..."}
```

### 5.2 响应特征

成功响应包含：

1. `access_token`
2. `refresh_token`（可轮换）
3. `expires_in: 3600`
4. `user` 对象

### 5.3 关键限制

1. 新拿到的 `Supabase access_token` 可访问 `rest/v1/*`。
2. **不能直接访问** `https://beta.magai.co/api/chat`（会 401）。
3. 必须经由 `next-action 40cd...` 再换 Magai 短 JWT。

## 7. 当前可复现实验最小序列

1. 使用有效登录态 cookie 调 `next-action=40cd...` 获取短 JWT。
2. 带短 JWT 调 `POST /api/chat`，确认 NDJSON 流返回。
3. 观察后续 `PATCH /rest/v1/spark` 中 `word_usage` 回填。
4. 如 session 过期，先走 `auth/v1/token?grant_type=refresh_token` 恢复 Supabase 会话，再重复步骤 1。

---

## 8. 注意事项

1. `/api/chat` 是业务层鉴权，不等同 Supabase REST 鉴权。
2. 短 JWT TTL 很短（约 5 分钟），过期后需重新走 `next-action 40cd...`。
3. 抓包中多条 token 已过期，复现实验需使用最新 token/cookie。
