# 批量注册功能 - 开发状态总结

## 已完成

### 核心程序：`apps/server/src/register.ts`

已完成 5 步注册流程的自动化实现：

1. **Signup Preflight** — 提交邮箱+密码到 next-action `407e2e7e...`
2. **Email Check** — 检查邮箱可用性，next-action `40317846773...`
3. **Create User** — 创建用户，next-action `60f8e7ce0...`
4. **Password Sign-in** — 通过 Supabase `grant_type=password` 获取 `access_token` + `refresh_token`
5. **Warm-up** — 调用 user boot + next-token 稳定 cookie（best-effort）

### 功能特性

- 自动生成随机邮箱/密码（`frank0000xxx@outlook.com`）
- Cookie 链累积（跨 5 步 Set-Cookie 合并）
- 多 Worker 并发注册（`--concurrency`）
- 请求间隔控制（`--delay` ms）
- Dry-run 模式（`--dry-run`）
- 结果导出（`--export`）
- 自动合并到账号池（`accounts.json`）

### CLI 参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--count` / `-n` | 注册数量 | 1 |
| `--concurrency` / `-c` | 并发数 | 1 |
| `--delay` | 请求间隔(ms) | 800 |
| `--domain` | 邮箱域名 | outlook.com |
| `--prefix` | 邮箱前缀 | frank |
| `--email` | 固定邮箱 | 随机生成 |
| `--password` | 固定密码 | 随机生成 |
| `--dry-run` | 演练模式 | false |
| `--export` | 导出路径 | 无 |
| `--skip-merge` | 不合并到账号池 | false |

### 运行命令

```bash
pnpm --filter @apps/server register --count 1
```

### 测试结果

- Dry-run 测试通过
- 单账号真实注册测试通过：成功注册 `frank4446cfd@outlook.com`，获取到 `userId` + `refreshToken`

## 已知问题

### 路径解析 Bug（未修复）

当通过 `pnpm --filter @apps/server exec tsx src/register.ts` 运行时，cwd 是 `apps/server/`，但 `MAGAI_ACCOUNTS_FILE` 和 `MAGAI_REGISTER_LOG` 默认值使用了相对于仓库根目录的路径 `apps/server/accounts.json`，导致文件被写入 `apps/server/apps/server/accounts.json`（双层嵌套）。

**修复方向**：在 `register.ts` 中将默认路径改为 `./accounts.json` 和 `./registered.json`，或使用 `path.resolve(import.meta.url, ...)` 动态计算。

### 未测试项

- 并发注册（`--concurrency > 1`）
- 大批量注册
- 邮箱域名切换
- 与主代理服务的账号池兼容性（生成的 `accounts.json` 格式需确认完全匹配 `index.ts` 中的 `bootstrapAccounts()` 预期格式）
